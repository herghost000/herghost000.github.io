OpenGL 之摄像机与观察空间

OpenGL本身没有摄像机(Camera)的概念，但我们可以通过把场景中的所有物体往相反方向移动的方式来模拟出摄像机，产生一种我们在移动的感觉，而不是场景在移动。

  1. 我们讨论摄像机/观察空间(Camera/View Space)的时候，是在讨论以摄像机的视角作为场景原点时场景中所有的顶点坐标：
  2. 观察矩阵把所有的世界坐标变换为相对于摄像机位置与方向的观察坐标。
  3. 要定义一个摄像机，我们需要它在世界空间中的位置、观察的方向、一个指向它右测的向量以及一个指向它上方的向量。

摄像机位置

获取摄像机位置很简单。摄像机位置简单来说就是世界空间中一个指向摄像机位置的向量。

摄像机方向

  1. 我们需要一个向量是摄像机的方向，这里指的是摄像机指向哪个方向。
  2. 现在我们让摄像机指向场景原点：(0, 0, 0)。
  3. 用场景原点向量减去摄像机位置向量的结果就是摄像机的指向向量。
  4. 摄像机指向自身z轴负方向，但我们希望方向向量(Direction Vector)指向摄像机自身z轴正方向。
  5. 如果我们交换相减的顺序，我们就会获得一个指向摄像机自身正z轴方向的向量：

方向向量(Direction Vector)并不是最好的名字，因为它实际上指向从它到目标向量的相反方向（译注：注意看前面的那个图，蓝色的方向向量大概指向z轴的正方向，与摄像机实际指向的方向是正好相反的）。

右轴

  1. 我们需要的另一个向量是一个右向量(Right Vector)，它代表摄像机空间的x轴的正方向。
  2. 为获取右向量我们需要先定义一个上向量(Up Vector)。
  3. 把上向量和第二步得到的方向向量进行叉乘。
  4. 两个向量叉乘的结果会同时垂直于两向量，因此我们会得到指向x轴正方向的那个向量
  5. 如果我们交换两个向量叉乘的顺序就会得到相反的指向x轴负方向的向量
