42.0、内存分区介绍

C C 2012/10/23

本文于 2845 天之前发表，文中内容可能已经过时。

在程序没有执行前，有几个内存分区已经确定，分区虽然确定，但未加载到内存，程序运行时才会加载到内存
  1. text（代码区）：只读，函数
  2. data（全局初始化数据区/静态数据区）：初始化数据，全局变量，static变量，文字常量区（只读）
  3. bss（未初始化数据区）：未初始化的数据，全局变量，static变量
程序运行额外加载2个区
  1. stack（桟区）：函数参数，返回地址，调用上下文、普通局部变量，自动管理内存
  2. heap（堆区）：手动申请内存，手动释放，整个程序结束系统自动回收
  • 程序可执行代码存放于.text 段
  • 静态存储区通常指程序中的.bss 和.data 段
  • 动态空间为堆中的空间
  • 局部变量所占空间为栈上的空间
  • 桟区递减，桟区数组递增
  • 堆区递增，堆区数组递增
